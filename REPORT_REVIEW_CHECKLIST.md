# Impact Report 可读性评审清单

> 目标：让非工程人员也能“敢于审批”。  
> 使用方式：请业务/数据负责人阅读报告并逐项确认。

---

## 1) 基本信息清晰
- [ ] 报告明确说明 **指标名称**  
- [ ] 报告明确说明 **版本对比（A/B）**  
- [ ] 风险等级（L1/L2/L3）明确且可理解  
- [ ] 变更内容有一句话摘要  

## 2) 影响范围可理解
- [ ] 影响范围按对象类型分组（指标/实体/维度/属性/映射）  
- [ ] 影响列表不夸大、不遗漏  
- [ ] 影响范围能对应业务逻辑理解  

## 3) 证据链可复现
- [ ] 报告包含审计查询或证据入口  
- [ ] 报告包含样例查询集（Before/After）  
- [ ] 证据可执行并能回放  

## 4) 建议动作可执行
- [ ] L1/L2/L3 对应动作清楚  
- [ ] 如果需要审批，明确审批角色  
- [ ] 如果需要回滚，给出回滚建议  

## 5) 语言与表达
- [ ] 术语一致，不混乱  
- [ ] 内容不过度技术化  
- [ ] 能被非工程读者理解
